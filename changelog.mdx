- Introduced wireless configuration endpoint for In-Place Extensometer (IPX) via `https://api.ospreymeasurement.systems/v2/config`.

<Update label="2024-09-10" description="v2.0.0" tags={["feature", "breaking"]}>
## New Features

- Real-time dashboard integration at `https://dashboard.ospreymeasurement.systems` with live OTS and IPX visualizations.
- API v2 endpoints for automated data export: `POST /v2/export` with JSON payloads.

## Breaking Changes

- Updated authentication to require `{API_KEY}` header instead of query params.
- OTS firmware requires recalibration after upgrade.

## Bug Fixes

- Resolved IPX strain measurement drift under thermal expansion.
</Update>

## Upgrade Process

Follow these steps to update your Osprey devices and documentation.

<Steps>
  <Step title="Check Compatibility" icon="check-circle">
    Verify your hardware revision matches the firmware requirements.
    
    | Product | Required Firmware | Hardware Rev |
    |---------|-------------------|--------------|
    | OTS     | v2.0+             | A or B       |
    | IPX     | v1.3+             | B            |
  </Step>
  
  <Step title="Download Firmware" icon="download">
    <Tabs>
      <Tab title="OTS">
        Download from `https://ospreymeasurement.systems/support/ots-firmware-v2.1.0.zip`.
      </Tab>
      <Tab title="IPX">
        Download from `https://ospreymeasurement.systems/support/ipx-firmware-v2.1.0.zip`.
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Flash Firmware" icon="upload">
````bash
# Connect via USB and run:
osprey-flash --device /dev/ttyUSB0 --firmware ots-v2.1.0.bin
````
  </Step>
  
  <Step title="Verify Update" icon="settings">
    Check logs at `https://api.ospreymeasurement.systems/v2/status`.
  </Step>
</Steps>
